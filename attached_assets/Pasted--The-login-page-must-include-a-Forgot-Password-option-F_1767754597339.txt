

The login page must include a “Forgot Password” option.

Forms must be user-friendly, responsive, and clearly labeled.

Authentication Field Security & Validation

Password requirements:

Minimum of 8 characters

Must include:

At least one uppercase letter

At least one lowercase letter

At least one number

At least one special character

A password strength indicator must be displayed (e.g., Weak / Strong).

Weak passwords are not accepted.

The new password must not be the same as the previous password.

The reset form must include:

New Password

Confirm Password

Forgot Password & OTP Verification

When a user selects Forgot Password:

A 6-character alphanumeric OTP code is sent to the user’s registered email.

OTP behavior:

The OTP expires after 2 minutes if unused.

If expired or not received, the user can click Resend Code to receive a new OTP.

OTP input feedback:

If the OTP is incorrect, the input field turns red and shakes to indicate an error.

If the OTP is correct, a confirmation dialog is shown before proceeding.

Password Reset Flow

Upon successful OTP verification:

The user confirms the password reset action.

The password is updated securely.

The user is redirected to the login page.

Password reset logic

Login & Registration Functionality

Fully functional login and registration workflows.

Proper error handling and validation feedback.

User Profile Access

Authenticated users can access their profile after successful login.

Button Loading Spinner

All authentication-related actions (login, register, OTP verification, password reset) must display a loading spinner while processing to prevent multiple submissions.
