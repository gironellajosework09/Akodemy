📌 Prompt: Competency Calculation, Retake Penalty, and Badge Persistence Logic

Act as a senior full-stack developer and system architect for a gamified learning platform.

We need to refine the competency and progression logic of the system with the following rules and constraints:

1️⃣ Competency Calculation Rules

Each challenge must explicitly define which competency/competencies it targets (e.g., problemSolving, syntax, logic, algorithms, etc.).

Competency scores must be dynamic and performance-based, not permanently locked once increased.

2️⃣ Retake Penalty Logic

If a student retakes a challenge and submits an incorrect solution, the system must:

Decrease the corresponding competency score, even if the student previously solved the challenge correctly.

Apply the decrease only to the competency tied to that challenge, not globally.

Competency should always reflect the most recent performance trend, not just historical success.

3️⃣ Correct Submission Behavior

If a challenge is retaken and solved correctly, competency may:

Increase again, or

Restore previously lost competency,

But must respect predefined caps or diminishing returns (if implemented).

4️⃣ Badge Persistence Rule

Badges are permanent once claimed or achieved.

Badge status must never be revoked, even if:

The student later fails the same challenge

Competency score decreases after a retake

Badge logic must be fully decoupled from competency recalculation after acquisition.

5️⃣ Data Model & System Requirements

Challenges must store:

Target competency/competencies

Difficulty level

Weight or impact value

Competency updates must be:

Logged per submission attempt

Auditable (track increases, decreases, and reasons)

Badge achievements must be stored as immutable records.

6️⃣ UX / Transparency

Students should receive clear feedback explaining:

Why competency increased or decreased

That badges remain permanent once earned

The system should prevent confusion between competency score and badge ownership.

7️⃣ Edge Cases

Multiple failed attempts should progressively reduce competency but never below a defined minimum.

Bulk re-evaluation (e.g., regrading) must not revoke badges.

Competency recalculation should remain consistent across different devices and sessions.

🎯 Goal:
Ensure that competencies accurately reflect current mastery, while badges represent milestone achievements that remain permanent.
